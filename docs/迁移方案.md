## 三、技术选型

### 3.1 框架

| 项目 | 现有 | 迁移后 |
|------|------|--------|
| 框架 | Vue 3.4 | uni-app Vue3（同一套语法） |
| UI 库 | Element Plus | uni-ui + 自定义组件 |
| 路由 | vue-router | pages.json |
| 状态管理 | 无 | 无（或 Vuex/Pinia，如果需要） |
| 图表 | ECharts | ECharts renderjs 方案 |
| 网络 | fetch / WebSocket | uni.request / uni.connectSocket |
| 录音 | MediaRecorder API | uni.getRecorderManager() |
| 定位 | navigator.geolocation | uni.getLocation() |
| TTS | speechSynthesis API | uni.createInnerAudioContext() + 后端 TTS |

### 3.2 后端部署方案

**问题：** Python FastAPI 无法在手机本地运行，需要远程后端。

**推荐方案：** 局域网 PC 部署
- 在实训教室的 PC 上运行 FastAPI 后端（uvicorn 0.0.0.0:8000）
- 手机和 PC 连同一 WiFi
- 手机 App 连接 `http://<PC_IP>:8000`

**备选方案：** 云服务器部署
- 如果教室 WiFi 不稳定，可部署到免费云服务（如 Railway/Render）
- 手机通过公网 IP 访问后端

### 3.3 MediaPipe 感知层

**关键问题：** MediaPipe .task 模型文件在移动端能否加载？

**方案 A（推荐）：感知层留在 PC 后端**
- 手机 App 只负责 UI 展示和用户交互
- 摄像头画面由 PC 后端采集 + 推流到手机
- 手机通过 WebSocket 接收实时数据

**方案 B（备选）：移动端原生感知**
- 用 uni-app 原生插件或 5+ 插件调用 Android Camera API
- 需要重写 MediaPipe 加载逻辑（工作量大）

## 五、分步实施计划

### 第 1 阶段：工程初始化（本周）

1. 下载安装 HBuilderX
2. 创建 uni-app Vue3 项目
3. 配置 `manifest.json`：
   ```json
   {
     "app-plus": {
       "modules": {
         "Camera": {},
         "Speech": {},
         "Geolocation": {}
       },
       "distribute": {
         "android": {
           "permissions": [
             "<uses-permission android:name=\"android.permission.CAMERA\"/>",
             "<uses-permission android:name=\"android.permission.RECORD_AUDIO\"/>",
             "<uses-permission android:name=\"android.permission.ACCESS_FINE_LOCATION\"/>",
             "<uses-permission android:name=\"android.permission.INTERNET\"/>"
           ]
         }
       }
     }
   }
   ```
4. 配置 `pages.json` 基础路由
5. 编译验证 → 能出 APK，能在手机上打开

## 六、技术难点与风险

| 难点 | 影响 | 缓解措施 |
|------|------|---------|
| Element Plus 在移动端不可用 | 所有 UI 组件需替换 | 用 uni-ui + 自定义 CSS |
| ECharts 在 uni-app 中需 renderjs | GaugePanel 迁移复杂 | 用 renderjs 方案或 SVG 替代 |
| MediaPipe .task 移动端加载 | 摄像头感知不可用 | 感知层留 PC 后端，手机只做展示 |
| 后端部署位置 | 手机无法独立运行 | PC 局域网部署或云服务器 |
| WebSocket 长连接稳定性 | 实时数据可能断连 | uni.connectSocket + 自动重连 |
| 兰无代码提交权限 (view) | 代码无法合入主仓库 | 本地开发，成品后组长合入 |
